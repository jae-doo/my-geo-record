# 제목
## 최대 50 글자를 넘기지 않아야 한다.
#
## 타입 종류
### feat        새로운 기능 추가
### fix         버그 수정
### docs        문서 수정
### style       코드의 동작 변경이 없는 수정
### refactor    코드 리팩토링
### test        테스트 코드, 리팩토링 테스트 코드 추가
### chore       빌드 업무 수정, 패키지 매니저 수정 등 기능과 무관한 내용
### comment     주석 추가 및 변경
### remove      파일, 폴더 삭제
### rename      파일, 폴더명 수정
#
## subject 시작 동사
### Add         추가
### Remove      삭제
### Simplify    단순화
### Update      보완
### Implement   구현
### Prevent     방지
### Move        이동
### Rename      이름 변경
#############################
# 본문(선택 사항)
## *로 항목을 나타낸다.
## 한 줄당 72자 이내로 작성한다.
## 최대한 상세히 작성한다.
## 어떻게 보다는 무엇을 변경했는지 설명한다.
#############################
# 바닥글(선택 사항)
## 이슈 트래커 ID를 작성한다.
## 여러 개의 이슈 번호를 적을 때에는 쉼표로 구분한다.
#
## 유형 종류
### Fixes       이슈 수정 중
### Resolves    이슈를 해결한 경우
### Ref         참고할만한 이슈
### Related to  해당 커밋에 관련된 이슈 번호
